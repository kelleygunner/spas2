Simple QR Code

Simple QR Code provides a set of APIs to help you handle barcode with ease in your game. It supports both encoding and decoding with various of popular barcode formats.

Supported Code Formats:

Decoding
UPC-A, UPC-E, EAN-8, EAN-13, Code 39, Code 93, Code 128, ITF, Codabar, MSI, RSS-14, QR Code, Data Matrix, Aztec, PDF-417

Encoding
UPC-A, EAN-8, EAN-13, Code 39, Code 128, ITF, Codabar, Plessey, MSI, QR Code, PDF-417, Aztec, Data Matrix

Basic Usage

*All APIs are located in CielaSpike.Unity.Barcode namespace.

how to: Decode From An Image

  • Get a decoder. A decoder is kind of expensive so keep the reference and avoid creating it frequently.
var decoder = Barcode.GetDecoder();
  • Decode a Texture2D
var result = decoder.Decode(texture);
  • Get the result
if (result.Success) {
    // get type of the barcode which has been decoded.
    if (result.BarcodeType == BarcodeType.QrCode) {
        // text in the barcode.
        Debug.Log(result.Text);
        // or you want the raw bytes in the barcode.
        byte[] bytes = result.RawBytes;
    }
}

how to: Scan A Barcode

To scan barcode using WebCamTexture, you can use the WebCamDecoder script included in this asset’s package. WebCamDecoder runs on a background thread.

  • First, drag the script to a game object, then in another script, get the decoder from the game object. like.
var decoder = GetComponent<WebCamDecoder>();
  • After you’ve created an instance of WebCamTexture and get it playing.
decoder.StartDecoding(webCamTexture); // and the decoder will start to decode via the webcam.
  • In your Update method, pending the result like:
var result = decoder.Result;
if (result.Success) {
    // do your stuff
}
  • The decoder will stop its work when the script has been destroyed, or you can make it stop manually like:
decoder.StopDecoding();

If you don’t want to use WebCamTexture and get camera data from another source. Barcode decoder also support decoding from a pixel buffer. When you managed to get the camera data in pixels:

var decoder = Barcode.GetDecoder();

// decode from a Color32[] pixel buffer with specified width & height
var result = decoder.Decode(pixels, width, height);

how to: Encode String Into A Barcode

  • Get an encoder with a type that you want it to encode with. It’s not as expensive as a decoder, but avoid creating it repeatedly is still a good practice.
 var encoder = Barcode.GetEncoder(BarcodeType.QrCode);

 // you can change the type so you need not to create many encoders.
 encoder.BarcodeType = BarcodeType.Pdf417;
  • Encode a string.
var result = encoder.Encode("ENCODE ME");
// or you want to specify the size.
var result = encoder.Encode("ENCODE ME", 256, 256);
  • Read from the result.
if (result.Success) {
    // get a black-white texture.
    Texture2D code = result.GetTexture();
    // get a dot matrix.
    DotMatrix dots = result.Dots;

    //parse the dots.
    for (int x = 0; x < dots.Width; x++) {
        for (int y = 0; y < dots.Height; y++) {
            if (dots[x, y]) { // A black dot
                // :::
            }
        }
    }
} else {
    Debug.LogError(result.ErrorMessage);
}

*dots in a DotMatrix maps to pixels instead of data units of the barcode image. If you want it to map to the data units, specify zero width and height when generate the code.

how to: Encode Without Unity

The CielaSpike.Unity.Barcode.dll assembly can be used to generate barcode images in PNG format without opening Unity.

When you want to create a PNG file from an encoding result. Try:

byte[] pngData = result.GetPngData();

Then you can save the byte array in a file like:

System.IO.File.WriteAllBytes("a.png", pngData);

Note that you also need to reference UnityEngine.dll assembly in your code generation project.

There is a zip file contains a sample C# console program which generates QR codes. Note that you should reference the DLL file in the zip package.

Advanced Usage

Get additional information in the barcode

Some barcode contains additional metadata. To get it from a decode result, use the TryGetMetadata method.

object metadata;
if (result.TryGetMetadata(MetadataType.ErrorCorrectionLevel, out metadata)) {
    string ecLevel = (string)metadata;
    // :::
}

Supported metadata types are:

public enum MetadataType
{
    /// <summary>
    /// Denotes the likely approximate orientation of the barcode in the image. This value
    /// is given as degrees rotated clockwise from the normal, upright orientation.
    /// For example a 1D barcode which was found by reading top-to-bottom would be
    /// said to have orientation "90". This key maps to an integer whose
    /// value is in the range [0,360).
    /// </summary>
    Orientation,

    /// <summary>
    /// 2D barcode formats typically encode text, but allow for a sort of 'byte mode'
    /// which is sometimes used to encode binary data. While <c>DecodeResult</c> makes available
    /// the complete raw bytes in the barcode for these formats, it does not offer the bytes
    /// from the byte segments alone.
    /// This maps to a <c><![CDATA[IList<byte[]>]]></c> corresponding to the
    /// raw bytes in the byte segments in the barcode, in order.
    /// </summary>
    ByteSegments,

    /// <summary>
    /// Error correction level used, if applicable. The value type depends on the
    /// format, but is typically a String.
    /// </summary>
    ErrorCorrectionLevel,

    /// <summary>
    /// For some periodicals, indicates the issue number as an integer.
    /// </summary>
    IssueNumber,

    /// <summary>
    /// For some products, indicates the suggested retail price in the barcode as a
    /// formatted string.
    /// </summary>
    SuggestedPrice,

    /// <summary>
    /// For some products, the possible country of manufacture as a string denoting the
    /// ISO country code. Some map to multiple possible countries, like "US/CA".
    /// </summary>
    PossibleCountry,

    /// <summary>
    /// For some products, the extension text.
    /// </summary>
    UpcEanExtension,

    /// <summary>
    /// If the code format supports structured append and
    /// the current scanned code is part of one then the
    /// sequence number is given with it.
    /// </summary>
    StructuredAppendSequence,

    /// <summary>
    /// If the code format supports structured append and
    /// the current scanned code is part of one then the
    /// parity is given with it.
    /// </summary>
    StructuredAppendParity,

    /// <summary>
    /// PDF417-specific metadata.
    /// </summary>
    Pdf417ExtraMetadata,

    /// <summary>
    /// Unspecified, application-specific metadata. Maps to an unspecified object.
    /// </summary>
    Other
}

Specify Options To Decoder & Encoder

The decoder which is of type IBarcodeDecoder, and the encoder which is a IBarcodeEncoder both have an Options property to make extra control of the decoding/encoding process. You can pass an option when you call Barcode.GetEncoder and Barcode.GetDecoder, or change it later.

DecodeOptions decodeOptions = new DecodeOptions() {
    AutoRotate = false,
    PossibleTypes = new[] { BarcodeType.QrCode, BarcodeType.Pdf417 }
};

var decoder = Barcode.GetDecoder(decodeOptions);

EncodeOptions encodeOptions = new EncodeOptions() {
    Margin = 5
};

var encoder = Barcode.GetEncoder(BarcodeType.QrCode, encodeOptions);

Decode Options

  • AutoRotate:
    Try rotate the texture when decoding, default true.

  • TryHarder:
    Try look deeper into the texture, which might be a little slow, default false.

  • TryInverted:
    Try recognize mirror inverted code, default true.

  • CharacterSet:
    Text encoding of the code, default ASCII.

  • PossibleTypes:
    Image is known to be of one of a few possible types.

Encode Options

  • Margin:
    Margin of the code image to be generate.

  • Width/Height:
    Default image width and height of generated images.

The EncodeOptions class have several child classes correspond to specific code formats.

QR Code

var encoder = Barcode.GetEncoder(BarcodeType.QrCode,
    new QrCodeEncodeOptions() {
        ECLevel = QrCodeErrorCorrectionLevel.H
    });

Supported options are: EC level, disable ECI.

PDF417

var encoder = Barcode.GetEncoder(BarcodeType.Pdf417,
    new Pdf417EncodeOptions() {
        Compact = true
    });
}

Supported options are: use compact, compaction type, PDF417 dimensions.

Aztec Code

var encoder = Barcode.GetEncoder(BarcodeType.Aztec,
    new AztecEncodeOptions() {
        Layers = 3
    });

Supproted options are: error correction, layers.

DataMatrix

var encoder = Barcode.GetEncoder(BarcodeType.DataMatrix,
    new DataMatrixEncodeOptions() {
        EncodeMode = DataMatrixEncodeMode.Base256
    });

Supproted options are: data matrix encode mode, max/min size, symbol shape.

Code 128

var encoder = Barcode.GetEncoder(BarcodeType.Code128,
    new Code128EncodeOptions() {
        ForceCodesetB = true
    });

Supproted options are: force using codeset B.
